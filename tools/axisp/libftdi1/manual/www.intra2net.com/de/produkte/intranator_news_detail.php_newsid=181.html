Intra2net Groupware Client 2.3.0

Anzeigen von mehreren Server-Konten in einer Ordnerstruktur (09.03.2015)

Neue Funktionen

  • Unterstützung von verschiedenen Server-Konten in einer Groupware-Datendatei. Durch diese Funktion können Groupware-Daten über Server-Grenzen hinweg genauso verwendet werden wie freigegebene Ordner
  • Der gelesen Status kann auf Wunsch mit anderen Benutzern geteilt werden. Diese Option kann pro Ordner im Freigabe-Dialog aktiviert werden
  • Wenn der letzte Bearbeiter bekannt ist, wird er als Kategorie in der Form [Geändert von: IMAP-LOGIN-NAME] im Objekt angezeigt
  • Neue Funktion "Ordnerliste aktualisieren" im Kontextmenü der Outlook-Ordnerliste
  • In Outlook 2010 und 2013 wird das Groupware Client Menü ab sofort im Menüband angezeigt
  • Beim Anlegen eines Server-Kontos im Groupware Client können Daten von bereits konfigurierten E-Mail-Konten übernommen werden

Verbesserungen

  • Bei der Profileinrichtung unter Outlook 2013 erhält die Outlook-IMAP-Datendatei nicht mehr zusätzlich die Markierung als Standard-Datendatei
  • Steuerzeichen werden beim Synchronisieren zum Server kodiert
  • Unterordner, die in einer zurückgespielten Kontosicherung nicht mehr enthalten sind, wurden fälschlicherweise auf dem Server wieder als leere Ordner angelegt
  • Arbeitsspeicherverbrauch reduziert
  • Weitere Detailverbesserungen
